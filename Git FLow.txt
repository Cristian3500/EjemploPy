Ramas de función en Git.

1. 	git checkout main 					--- Actualizacion del main en local.
	git fetch origin 						--- Se busca origen.
	git reset --hard origin/main 				--- Deseche todos mis cambios preparados y no preparados, olvídese de todo en mi rama local actual y hágalo exactamente igual que origin/master.

2.	git checkout -b new-feature				--- Crea una rama nueva. Se extrae una rama llamada "new-feature" a partir de la main, "-b" ordena a Git que la cree en caso de que esta no exista.

	--- Se agregan los archivos a la nueva rama.
3.	git status							
	git add <some-file>
	git commit
